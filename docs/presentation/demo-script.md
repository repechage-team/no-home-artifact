# NoHome 시연 시나리오 (Demo Script)

> 발표 16번 슬라이드용 라이브 데모 진행 가이드.
> 목표: 2~3분 안에 "검색 → 지도 → AI 조작 → 회원"의 end-to-end 흐름을 끊김 없이 보여준다.

---

## 사전 준비 체크리스트 (발표 직전)

- [ ] Docker Compose 기동 확인: `docker compose ps` (backend/frontend/mysql 모두 Up)
- [ ] Frontend 접속: `http://localhost:5173` HTTP 200
- [ ] Backend Health: `http://localhost:8080/api/health` 정상
- [ ] 환경변수 확인: `PUBLIC_DATA_SERVICE_KEY`, `PUBLIC_DATA_APT_RENT_SERVICE_KEY`, `SSAFY_GMS_API_KEY`, `KAKAO_MAP_API_KEY`(=`VITE_KAKAO_MAP_API_KEY`) 모두 설정됨
- [ ] **Kakao Web 플랫폼 도메인**에 시연 origin 등록 확인 (미등록 시 지도 401 — 과거 이슈)
- [ ] 시연용 계정 1개 미리 가입 (예: `demo@nohome.dev`) — 라이브 가입 실패 대비
- [ ] 시연 지역 데이터 사전 적재: 예) 강남구 최근 1~2개월 매매/전월세 (자동 적재 지연 방지)
- [ ] **백업 자료 준비**: 전체 흐름 화면 녹화 영상 + 단계별 스크린샷 (네트워크/지도 장애 대비)

> 자동 임포트는 첫 검색 시 외부 API 호출로 수 초 지연될 수 있음. 시연 지역은 반드시 미리 한 번 검색해 DB에 적재해 둘 것.

---

## 시연 흐름 (4단계)

### STEP 1 — 지역 + 거래유형 검색 (약 30초)
1. 거래유형 **매매** 선택
2. 지역: 서울특별시 → **강남구** → (법정동 자동 로드 확인) 동 선택
3. 거래월 범위 설정 (사전 적재한 월)
4. **검색** 클릭
- **보여줄 것**: 좌측 결과 카드 목록 + 우측 지도 마커가 동시에 뜨는 화면
- **멘트**: "검색하면 목록과 지도가 함께 표시됩니다."

### STEP 2 — 가격 필터 조정 (약 20초)
1. 가격 이중 슬라이더로 상한/하한 조정
2. 결과가 좁혀지는 것 확인
- **보여줄 것**: 필터에 따라 결과 수·마커가 줄어드는 반응
- **멘트**: "거래유형에 맞는 가격 필터로 범위를 좁힐 수 있습니다."

### STEP 3 — AI 어시스턴트 화면 조작 ⭐ (약 60초, 하이라이트)
1. 우측 하단 챗봇 FAB 열기
2. 입력: **"월세로 보여주고 보증금 1억 이하만"**
3. (응답 대기 — 진행 표시 노출)
4. 화면 자동 변화 확인: 거래유형이 월세로 전환 + 보증금 필터 적용 + 검색 실행
5. 추가 입력 예시(시간 여유 시): **"다음 페이지 보여줘"** 또는 **"첫 번째 매물 지도에서 보여줘"**
- **보여줄 것**: 말 한마디로 필터·검색·페이지·지도가 움직이는 장면
- **멘트**: "데이터를 묻는 걸 넘어, 말로 UI를 직접 조작합니다. 이게 저희의 차별점입니다."
- **주의**: LLM 응답은 약간의 지연이 있음 — 자연스럽게 멘트로 채우기

### STEP 4 — 회원 로그인 & 내 정보 (약 20초)
1. 상단 계정 메뉴 → 로그인 (사전 가입 계정)
2. 로그인 성공 후 내 정보 조회
- **보여줄 것**: 인증 후 보호 화면 접근
- **멘트**: "회원 인증까지 end-to-end로 동작합니다."

## 대체 시나리오 (장애 대비)

| 상황 | 대응 |
| --- | --- |
| 지도 마커 미표시 (Kakao 401/네트워크) | 백업 스크린샷으로 대체, "도메인 등록 이슈는 트러블슈팅에서 다룬 항목" 언급 |
| AI 응답 지연/실패 (GMS 장애) | 사전 녹화 영상 재생 또는 질문 모드 1건으로 축소 |
| 자동 임포트 지연 | 사전 적재 지역으로 즉시 검색 (지연 노출 회피) |
| 로그인 실패 | 사전 가입 계정 재확인, 실패 시 화면만 시연하고 스킵 |
