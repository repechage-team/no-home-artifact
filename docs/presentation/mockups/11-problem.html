문제 정의
1
공공데이터 API의 높은 진입장벽
국토부 실거래가 API는 XML 응답 + 법정동코드(LAWD_CD) + 계약년월(DEAL_YMD) — 비개발자가 직접 다루기 어렵다
2
흩어진 거래유형 데이터
매매 / 전세 / 월세가 서로 다른 API에 분산 (RTMSDataSvcAptTrade · RTMSDataSvcAptRent)
3
대화형 화면 조작 경험의 부재
기존 서비스는 데이터 질문엔 답해도, 대화로 필터·페이지·지도를 직접 조작해 주지는 않는다 — 결국 사용자가 일일이 만져야 한다
그래서 NoHome은 — 자연어 대화로 검색·필터·페이지·지도까지 조작하는 AI 어시스턴트
