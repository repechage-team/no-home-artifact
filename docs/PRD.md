# PRD.md

## 제품 요약

SSAFY Home은 공공데이터 기반 주택 정보 서비스다. 사용자는 동별 또는 아파트명별로 주택 실거래가를 검색하고, 필요한 주택 정보를 확인하며, 자신의 회원 정보를 관리할 수 있어야 한다.

이번 프로젝트의 중심 목표는 기존 SSAFY Home의 백엔드 구조를 Spring Boot와 MyBatis 기반으로 전환하고, Vue.js 화면이 REST API를 호출하는 구조로 과제 명세의 필수 기능인 실거래가 검색, 회원 관리, 로그인/로그아웃을 안정적으로 구현하는 것이다.

## 목표

- 사용자가 공공데이터 API를 몰라도 주택 실거래 정보를 쉽게 찾을 수 있게 한다.
- 동별 검색과 아파트명 검색을 제공한다.
- 회원 가입, 조회, 수정, 삭제, 로그인, 로그아웃을 제공한다.
- 공공데이터를 가공해 DB에 저장하고, DB 기반으로 빠르게 조회한다.
- 필수 기능을 먼저 검증한 뒤, 관심 지역이나 주변 정보 같은 추가 기능을 선택적으로 확장한다.

## 사용자

- 특정 동네의 주택 실거래가를 확인하려는 사용자.
- 아파트 이름으로 거래 이력을 비교하려는 사용자.
- 회원 가입 후 자신의 정보를 관리하려는 사용자.
- Spring + MyBatis 전환 결과를 확인하는 평가자 또는 개발자.

| ID | 요구사항 | 완료 기준 |
| --- | --- | --- |
| F701 | 주택 실거래가 정보 수집 | 아파트 매매, 아파트 전월세, 연립다세대 매매, 연립다세대 전월세 데이터를 DB에 적재할 수 있다. |
| F702 | 주택 실거래가 검색 | 동별 검색과 아파트명 검색으로 조건에 맞는 실거래가 목록을 조회할 수 있다. |

## 필수 화면 / 흐름

- 실거래 정보를 활용한 메인 페이지.
- 동별 실거래가 검색 결과 페이지. 검색 결과는 지도에도 표시한다.
- 아파트명별 실거래가 검색 결과 페이지. 검색 결과는 지도에도 표시한다.
- 회원 가입 페이지.
- 회원 정보 조회, 수정, 삭제 페이지.
- 세션 기반 로그인/로그아웃 흐름.

## 추가 기능 후보

| ID | 기능 | 가치 | 비용 / 리스크 |
| --- | --- | --- | --- |
| F703 | 관심 지역 관리 | 개인화된 진입점을 제공한다. | 회원과 지역 데이터를 연결하는 저장 구조가 필요하다. |
| F704-F705 | 주변 상권/업종 검색 | 주거지 선택에 도움이 된다. | 외부 데이터 적재, 업종 분류, 지도/검색 UI가 필요하다. |
| F706-F707 | 주변 환경 정보 검색 | 녹지, 폐수, 대기 배출 등 생활 환경 판단에 도움이 된다. | 데이터 해석과 지도 표시가 복잡할 수 있다. |
| F717 | 비밀번호 찾기 | 계정 사용성을 높인다. | 임시 비밀번호 또는 토큰 정책이 필요하다. |
| F711 | 공지사항 관리 | CRUD 범위가 명확하고 구현 난도가 낮다. | 부동산 서비스의 핵심 가치와는 거리가 있다. |

## 심화 / 보류 후보

- 주택 관련 뉴스 크롤링과 최근 뉴스 제공.
- 공유 게시판 등록, 수정, 삭제, 조회.
- 외국인 수, 고령자 수 등 동네 구성원 정보 제공.
- 동네 CCTV 설치 현황 검색.

## 비기능 요구사항

- 사용자는 웹 사이트에 대한 사전 지식이 없어도 쉽게 사용할 수 있어야 한다.
- 검색 결과는 반복 조회가 가능할 정도로 빠르게 응답해야 한다.
- 공공데이터 적재 과정에서 출처와 정확성을 유지해야 한다.
- 주요 화면은 일반적인 데스크톱과 모바일 브라우저 폭에서 사용할 수 있어야 한다.
- 데이터 변경 기능은 테스트 또는 반복 가능한 수동 검증으로 확인한다.

## Decision Board

아래 항목은 개발자가 확정한 제품 방향이다.

| 결정 | 확정안 | 남은 세부 결정 |
| --- | --- | --- |
| 공공데이터 처리 | API/파일 데이터를 DB에 먼저 적재하고, 서비스 검색은 DB를 우선 조회한다. | 초기 적재 지역/기간/주택 유형 범위, 미적재 요청 발생 시 부족분만 적재할지 또는 요청 범위 전체를 재적재할지 결정해야 한다. |
| 회원 삭제 정책 | 물리 삭제. | 삭제 전 연관 데이터 처리와 재가입 허용 정책을 구현 Sprint에서 확인한다. |
| 첫 추가 기능 | 주변 상권 지도. | 사용할 상권 데이터 출처, 업종 분류 범위, 지도 API 선택이 필요하다. |
| 지도 범위 | 실거래가 검색 결과도 지도에 표시한다. | 실거래가 데이터의 좌표 확보 방식과 좌표가 없는 항목의 표시 정책이 필요하다. |
| 인증 방식 | 세션 기반 로그인. | 세션 만료 시간, 로그인 필요 화면, 인터셉터 적용 범위를 구현 Sprint에서 확정한다. |
| 애플리케이션 구조 | Spring Boot 기반 REST API + Vue.js. 최종 제출물은 Vue 빌드 결과를 Spring Boot 정적 리소스에 포함한다. | 개발 중 Vue/Vite 프로젝트 위치와 빌드 산출물 복사 방식을 구현 Sprint에서 확인한다. |

## 현재 계획

MVP는 Spring Boot, Vue.js, REST API, DB 우선 실거래가 데이터, 세션 기반 로그인, 회원 물리 삭제, 실거래가 검색 지도 표시, 첫 추가 기능인 주변 상권 지도를 기준으로 진행한다. 최종 제출물은 Vue 빌드 결과를 Spring Boot 정적 리소스로 포함한다. 공공데이터의 초기 적재 범위와 미적재 요청 처리 방식은 M2 시작 전에 별도 결정한다.
